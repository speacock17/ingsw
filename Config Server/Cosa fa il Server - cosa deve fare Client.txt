-- UTENTI
-Server:
	- recupera i dati di un utente	
		- in base all'username			(controlla l' utente esiste)
		- in base al ruolo 	(restituisce l'elenco)
	
	- registra i dati di un nuovo utente		(controlla se un username è già in uso)
	
	- modifica i dati di un utente		(controlla se l' utente esiste)
		- nome
		- cognome
		- username				(controlla se il nuovo username è già in uso)
		- password
		- ruolo				
	
	- elimina interamente un utente 		(controlla se l' utente esiste)
	
	- effettua il login 
		.- controlla se l'username ricevuto dal client esiste
		.- recupera la password corrispondente
		.- confronta la password recuperata dal Database con quella ricevuta dal client
		.- se le password coincidono invia un messaggio di Successo, altrimenti di Errore
		

-Client:
	- effettua controlli sulle lunghezze delle stringhe e i caratteri inseriti
	- controlla che la password (nuova o per il login) rispetti determinati criteri
	- fa inserire un ruolo a scelta (ma sicuramente esistente dall'elenco dell'enumeration) e non scrivibile a mano libera
